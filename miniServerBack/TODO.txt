                            TESTER LE BACK

D'apres les explications de Damien.
Faire un mini serveur pour tester la connexion entre le front et le back.
Verifier que l'on recupere bien les bonnes informations que nous renvoie le back.
Faire un controller qui va contenir des objets en JSON qui contiendront des
informations fixes et les utiliser ensuite dans le front. 
Comme par exemple pour le login ou la liste de user:

	@Get("yooyoo")
	yooyoo() {
		return [
					{
						"user": "yooyoo1", // On mettra ici donc les informations a recuperer du back
						"avatar": "/default.jpg"
					},
					{
						"user": "yooyoo2",
						"avatar": "/default.jpg"
					},
					{
						"user": "yooyoo3",
						"avatar": "/default.jpg"
					}
				];
	}
    =>
    curl https://localhost:[PORT]/yooyoo
    =>
    Doit nous afficher les objets JSON contenus dans le controller. 


    // front -> back
    @Get("login")
	    login() { 
            return true; // ou return infos du login selon besoin
        }
	}
    // back -> front
        true

Pour faire fonctionner le controller, il faut utiliser l'app.module.ts et le module.ts.

Le back lui, va faire un mini testeur aussi. En envoyant des requetes pour voir si 
la connexion fonctionnerait. Car le front renvoie juste des requetes au back.


=> On devra donc faire une liste des informations dont on a besoin de recuperer du back avant.


PS: On doit faire le mini serveur pour pouvoir tester le lien avec le back
pendant que le back n'est pas encore fini et qu'il n'est pas encore possible 
de synchroniser le front et le back ensemble.
